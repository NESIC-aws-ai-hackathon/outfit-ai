# Unit of Work — Story Map

## テーマ適合性

> **「使うほど、自分で今日の自分を決めなくなる」**

本サービスは、朝の服選び・TPO判断・自己演出までAIに任せることで、ユーザーが自分で"今日の自分"を決めなくなる体験を提供する。AIは予定・天気・手持ち服をもとに"今日どう見られるべきか"を定義し、それを服装として出力する。

ここでいう「今日の人格」とは性格診断ではなく、予定・TPOからAIが定義する**外向きの印象設計**を指す。ユーザーは使い続けるほど、この印象設計の意思決定を放棄し、AIが決めた人格をそのまま着る状態に近づいていく。

---

## ユーザーストーリー（MVP）

以下のユーザーストーリーは、ハッカソンテーマ「人をダメにする」に適合する形で、MVPで実装する各機能に紐づけて定義する。

### US-001: 手持ち服登録（→ Unit 2: Closet）

**As a** 朝に服を選ぶのが面倒なユーザー,
**I want** AIがタグ付け・分類してくれる前提で手持ち服を登録しておきたい,
**So that** コーデを考える材料すらAI管理になり、自分で組み合わせを考える余地がなくなる。

### US-002: 予定の自動取得（→ Unit 3: Calendar）

**As a** 予定に合う服を自分で考えたくないユーザー,
**I want** 今日の予定がGoogle Calendarから自動で取り込まれてほしい,
**So that** 予定の確認すら不要になり、AIが文脈を勝手に理解してくれる。

### US-003: 天気情報の自動取得（→ Unit 4: Weather）

**As a** 天気を調べて服を変えるのが面倒なユーザー,
**I want** 天気情報が自動で取得され服装判断に使われてほしい,
**So that** 天気予報を見る判断すらAIに丸投げできる。

### US-004: 今日の人格定義（→ Unit 5: Outfit）

**As a** 自分で"今日の自分"を決めたくないユーザー,
**I want** AIが予定・天気・TPOから「今日の人格」を定義してほしい,
**So that** 自己演出・印象設計の意思決定を完全にAIに委ねられる。

### US-005: AI服装提案（→ Unit 5: Outfit）

**As a** 朝クローゼットの前で何も考えたくないユーザー,
**I want** AIが「今日の人格」に基づいた具体的な服装を提案してほしい,
**So that** 朝起きたら着るものが決まっている状態で1日を始められる。

### US-006: 選定理由・除外理由の表示（→ Unit 5: Outfit）

**As a** なぜその服が選ばれ、他の服が外されたのか気にはなるが自分では考えたくないユーザー,
**I want** AIがその服を選んだ理由と、他の服を除外した理由を確認できること,
**So that** 自分で判断を下さなくても「これでいい」と安心して委ねられる。

### US-007: 着用履歴管理（→ Unit 5: Outfit）

**As a** 同じ服ばかり着ていないか気にしたくないユーザー,
**I want** AIが着用履歴を管理し連続着用を自動で避けてほしい,
**So that** 着たものの記憶すらAI任せにでき、自分で覚えておく必要がなくなる。

### US-008: 再生成（→ Unit 5: Outfit）

**As a** 提案が気に入らないが自分で考え直したくないユーザー,
**I want** 「別の提案」ボタン一つで再提案してほしい,
**So that** 自分でコーデを組み直す労力なく、もう一度AIに丸投げできる。

### US-009: ユーザー認証（→ Unit 1: Auth）

**As a** 自分の手持ち服・予定情報・着用履歴を安全に管理したいユーザー,
**I want** アカウント登録・ログインにより個人単位でデータを保持したい,
**So that** 一度登録すれば何も準備せずAIに委ねるだけの状態が維持される。

## ユニット別機能マップ

### Unit 1: Auth
| 機能 | 優先度 | 説明 |
|------|--------|------|
| ユーザー登録 | 必須 | メール+パスワードによる新規登録 |
| ログイン | 必須 | JWT発行 |
| ログアウト | 必須 | セッション無効化 |
| パスワードリセット | 低 | メール送信によるリセット |

### Unit 2: Closet
| 機能 | 優先度 | 説明 |
|------|--------|------|
| 服一覧表示 | 必須 | フィルタ付き一覧 |
| 服登録（手動） | 必須 | フォーム入力 |
| 写真アップロード | 必須 | S3 + プリサインドURL |
| AIタグ付け（同期: 基本） | 必須 | 色・種別の即時推定 |
| AIタグ付け（非同期: 詳細） | 中 | 季節感・TPO適性等 |
| 服編集・削除 | 必須 | CRUD操作 |

### Unit 3: Calendar
| 機能 | 優先度 | 説明 |
|------|--------|------|
| Google OAuth連携 | 必須 | 読み取り専用スコープ |
| 今日の予定取得 | 必須 | 当日の予定一覧 |
| 予定の手動入力 | 必須 | フォールバック用 |
| トークンリフレッシュ | 必須 | 自動リフレッシュ |

### Unit 4: Weather
| 機能 | 優先度 | 説明 |
|------|--------|------|
| 天気情報取得 | 必須 | OpenWeatherMap API |
| 15分キャッシュ | 必須 | DynamoDBベース |
| 位置情報管理 | 必須 | デフォルト都市 + Geolocation上書き |

### Unit 5: Outfit Recommendation
| 機能 | 優先度 | 説明 |
|------|--------|------|
| コンテキスト統合 | 必須 | 天気+予定+服+履歴を統合 |
| AI提案生成 | 必須 | Bedrock呼び出し、「今日の人格」生成 |
| 提案結果表示 | 必須 | 今日の人格、提案された服装、選定理由、除外理由を表示 |
| 再生成 | 必須 | 「別の提案」リクエスト |
| 提案採用→着用記録 | 必須 | 着用履歴保存 |
| 連続着用回避 | 中 | 直前の着用を除外 |
