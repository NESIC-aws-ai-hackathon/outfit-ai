# Unit of Work — Story Map

## 機能要件とユニットのマッピング

| 機能要件 | ユニット | FE対応 | BE対応 | 備考 |
|---------|---------|--------|--------|------|
| **FR-001: ユーザー認証** | Unit 1: Auth | 登録・ログインページ | Auth Service | JWT管理、パスワードハッシュ化 |
| **FR-002: クローゼット管理** | Unit 2: Closet | クローゼットCRUD画面 | Closet Service | 画像アップロード、AIタグ付け(同期/非同期) |
| **FR-003: 日次コンテキスト入力** | Unit 3: Calendar + Unit 5: Outfit | 予定入力・Calendar表示 | Calendar Service + Outfit Service | Google Calendar連携 + 手動入力 |
| **FR-004: 天気情報取得** | Unit 4: Weather | 天気表示UI | Weather Service | OpenWeatherMap API, 15分キャッシュ |
| **FR-005: AI服装提案** | Unit 5: Outfit | 提案表示・再生成UI | Outfit Recommendation Service | Bedrock(Claude), コンテキスト統合 |
| **FR-006: 着用履歴管理** | Unit 5: Outfit | 「採用」ボタンUI | Wear History Service | 直前着用追跡、連続回避 |

## ユニット別機能マップ

### Unit 1: Auth
| 機能 | 優先度 | 説明 |
|------|--------|------|
| ユーザー登録 | 必須 | メール+パスワードによる新規登録 |
| ログイン | 必須 | JWT発行 |
| ログアウト | 必須 | セッション無効化 |
| パスワードリセット | 低 | メール送信によるリセット |

### Unit 2: Closet
| 機能 | 優先度 | 説明 |
|------|--------|------|
| 服一覧表示 | 必須 | フィルタ付き一覧 |
| 服登録（手動） | 必須 | フォーム入力 |
| 写真アップロード | 必須 | S3 + プリサインドURL |
| AIタグ付け（同期: 基本） | 必須 | 色・種別の即時推定 |
| AIタグ付け（非同期: 詳細） | 中 | 季節感・TPO適性等 |
| 服編集・削除 | 必須 | CRUD操作 |

### Unit 3: Calendar
| 機能 | 優先度 | 説明 |
|------|--------|------|
| Google OAuth連携 | 必須 | 読み取り専用スコープ |
| 今日の予定取得 | 必須 | 当日の予定一覧 |
| 予定の手動入力 | 必須 | フォールバック用 |
| トークンリフレッシュ | 必須 | 自動リフレッシュ |

### Unit 4: Weather
| 機能 | 優先度 | 説明 |
|------|--------|------|
| 天気情報取得 | 必須 | OpenWeatherMap API |
| 15分キャッシュ | 必須 | DynamoDBベース |
| 位置情報管理 | 必須 | デフォルト都市 + Geolocation上書き |

### Unit 5: Outfit Recommendation
| 機能 | 優先度 | 説明 |
|------|--------|------|
| コンテキスト統合 | 必須 | 天気+予定+服+履歴を統合 |
| AI提案生成 | 必須 | Bedrock呼び出し、「今日の人格」生成 |
| 提案結果表示 | 必須 | 人格+コーデ+理由+代替案 |
| 再生成 | 必須 | 「別の提案」リクエスト |
| 提案採用→着用記録 | 必須 | 着用履歴保存 |
| 連続着用回避 | 中 | 直前の着用を除外 |
